1.使用和安装第三方包（可能是框架，可能是库，也可能是插件），只需记住名字即可 2.一次配置，以后再任何地方，任何的设备，只需输入命令，会自动下载安装项目的依赖 package.json 每个项目的根目录下面，一般都有一个package.json文件，定义了这个项目所需要的各种模块(可能是一个文件，也可能是一个包)， 以及项目的配置信息（比如名称、版本、许可证等元数据）。 npm install命令根据这个配置文件，自动下载所需的模块，也就是配置项目所需的运行和开发环境。 开发环境 1.开发环境,代码都是源代码，没有经过压缩合并，并且会输出错误日志,通常这类的代码都放在 src 或者 source文件夹下 Development 2.生产环境,指的是，发布在线上的代码，这些代码通常是经过压缩合并的代码，通常放在 dist 或者 build 文件夹下 production 1.初始化package.json文件 npm init 2.安装模块 npm install [module name] 本地（项目）安装 npm install [module name] --save 本地（项目）安装，将该模块设置为生产环境依赖，会在package.json文件中的dependencies显示 npm install [module name] --save-dev 本地（项目）安装，将该模块设置为生产环境和开发环境依赖, 会在package.json文件中devDependencies显示 npm install [module name] -g 全局（电脑机器）安装，安装在这台电脑上 1.安装一个工具，在任何的项目或者文件夹里都可以使用 3.配置package.json的scripts字段 npm run <命令名> 属性名是命令名，属性值是命令 npm默认是从外国的服务器下载，所以有时安装模块会特别的慢 解决方案：设置下载的源为国内的淘宝镜像 npm config set registry https://registry.npm.taobao.org --global npm config set disturl https://npm.taobao.org/dist --global 4.在项目根目录下创建 .gitgnore 文件，push仓库的时候，会忽略不上传 如何安装第三方包 房间卡了圣诞节
